HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Common elements in HTML: text, image, header, list.
  • HTML is building the framework for the webpage.
  • Headings are listed in numerical order, like h1, h2, etc. The earlier numbers produce bigger headings, and the later ones are smaller headings.
  • Doctype declaration tells cpu what kind of document it is.
  • An attribute (defined w an equal sign) defines additional info about the element it is nested in.
  • Metadata tells our browser information about the webpage.
  • Semantic elements' names describe them.
  • ul is unordered list, ol is ordered list, li is list element.
  • img element contains info about images on webpage; src attribute defines location of image on webpage; alt attributes are important for assistive tech, describes function and appearance of image.

CSS

  • CSS (cascading style sheets) make the project more readable and appealing to viewer.
  • inline CSS apply to specific elements, each must be edited individually
  • internal style sheet applies rules for the entire file, located in the head section.
  • external style sheet is most common, links external files w CSS sheet to the base file; file is linked using a link element in the HTML head section
  • CSS RULE) selector {property:value}; brackets indicate declaration block, with each pair of property and value being a declaration.
  • CSS Box Model) content is surrounded by padding, which is surrounded by border, which is surrounded by margin
  • * is wild card selector that applies rules to all elements on page
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A class attribute can give properties to all HTML elements assigned the class selector.
  • linking an external CSS: go to head section of html to add
  • make sure selector is assigned to the correct elements in the CSS and in the HTML

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • to sync changes in git repository:1)make sure you are in the correct directory (git status). 2)git add -A to commit files to working branch. 3)git commit to commit changes (-m to add a message). 4)git pull origin main to pull in main branch. 5)git push origin (branch) to push changes to main branch. 6)create and complete pull request in github, w/ main branch as base and (branch) as compare. 7)navigate to local main branch (git checkout main) and update local branch w/ git pull origin main
  • checkout git cheatsheet for more git shortcuts

JavaScript

  • JS is added near end of body section because adding it earlier can create issues.
  • variable: container that stores values
  • JS is case sensitive
  • use "let" to declare a variable
  • use structure variable = value to give it a value
  • variable types: string, boolean, number, array, object
  • can use comments by enclosing them in /* */
  • Operator produces results based on two variables.
  • operator types: add, subtract, multiply, divide, assignment, strict equality
  • conditionals can dictate what code is run (if then statements)
  • with if statements, code will only execute if it is true
  • cpu will only execute truthy in conditionals
  • falsy:0,-0,"",0n,null,undefined,NaN, false
  • functions: followed by parantheses and have multiple uses
  • events listen for activity in a browser
  • null is absence of value
  • BigInt can accurately store more integers than number type
  • Strings should only be used for textual data.
  • Objects are collections of properties; data property and accessor properties.
  • control flow:order in which cpu executes code in script
  • JS is read from top to bottom, order matters!
  • array: single variable that holds group of data
  • strings need to have quotation marks
  • index is number assigned to data in array (starting at 0)
  • for loops perform tasks on all items in an array
